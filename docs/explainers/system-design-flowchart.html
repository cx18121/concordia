how every piece connects

Read it as flows: a human logs in and verifies once, gets a wallet, deposits into the Vault, and votes through Voting — which reads capital from the Vault and accuracy from Reputation. Winners go to the Portfolio, which buys them on Uniswap. Chainlink CRE feeds real prices and scores back in. Rewards flow back to the best voters.

2 · The weekly cycle, step by step

the same picture, but over time

This is the temporal view — which contract talks to which, in order, across one full week. The numbers match the six cycle stages: Subscribe → Snapshot → Allocate → Select → Execute → Resolve.

sequenceDiagram
  autonumber
  actor U as User / Agent
  participant W as Wallet (Dynamic)
  participant V as Vault
  participant G as Voting
  participant R as Reputation
  participant P as Portfolio
  participant X as Uniswap v4
  participant K as Chainlink CRE
  participant F as Fees & Rewards

  Note over U,F: ① Subscribe — money in, shares out
  U->>W: approve deposit
  W->>V: deposit USDC (at cycle boundary)
  V-->>U: mint shares

  Note over U,F: ② Snapshot — freeze voting power
  G->>V: read capital share
  G->>R: read accuracy share
  G->>G: power = 50% capital + 50% accuracy

  Note over U,F: ③ Allocate — cast votes
  U->>G: weighted allocation across stocks

  Note over U,F: ④ Select — tally winners
  G->>P: top-5 by weight, 40% cap, renormalized

  Note over U,F: ⑤ Execute — buy the basket
  P->>X: swap USDC → stock tokens
  X-->>P: stock tokens (NAV now live)

  Note over U,F: ⑥ Resolve — score, smooth, pay
  K->>P: post fresh prices / NAV
  K->>R: update accuracy (EWMA α≈0.25)
  P->>F: performance vs S&P
  F-->>U: rewards to winning voters
  R-->>G: feeds next week’s snapshot
